Polymer

Polymer is a library for building web apps from components using cutting-edge technologies.

Opinionated, backed by Google.

What is it

In Polymer, everything is an element. Building apps is about building custom elements – and reusing existing ones.

Custom element

Self-contained component ready to use:

  • button
  • login form
  • modal popup
  • etc...

Custom elements have their own styles and their own functionalities.

They can be controlled through attributes and events.

Don't worry, you can still style any element on your page.

Polymer elements

Polymer has a set of Core elements available, ranging from core-xhr to core-tooltip.

The idea is that you can use elements to create new elements, e.g. fetch data with core-ajax in your x-weather element.

There are also experimental UI and non-UI elements in the Polymer labs.

Using elements

  1. bower install --save Polymer/polymer
  2. bower install --save Polymer/core-elements
  3. Use the elements

Getting started / Creating elements

elements/my-element.html


<link rel="import" href="../bower_components/polymer/polymer.html">

<polymer-element name="my-element" noscript>
  <template>
    <span>Hello from <b>my-element</b>. This is my Shadow DOM.</span>
  </template>
</polymer-element>

index.html


<!DOCTYPE html>
<html>
  <head>
    <!-- 1. Load platform support before any code that touches the DOM. -->
    <script src="bower_components/platform/platform.js"></script>
    <!-- 2. Load the component using an HTML Import -->
    <link rel="import" href="elements/my-element.html">
  </head>
  <body>
    <!-- 3. Declare the element by its tag. -->
    <my-element></my-element>
  </body>
</html>

A bit on Shadow DOM

Each element has its own Shadow DOM.

Shadow DOM encapsulates components but does not prevent manipulation.

Styles don't pass the Shadow boundary by default, but you can target Shadow DOMs using the ::shadow pseudo-element or the /deep/ combinator.
